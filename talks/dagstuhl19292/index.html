An Open-Source Framework for Crowdsourced Mobile Visualization Experiments

RangesOnMobile, MobileTrendVis experimental applications share common infrastructure:

· Node.js web application deployed on Azure
· Custom event logging with Azure Application Insights
· CSS media queries to enforce mobile-only
· Sever-side condition assignment & re-balancing
· Handlers for orientation changes & interruptions

Application developer to focus on:

· Isolated chart component
· Defining trial logic
· Trial bank initialization

Mobile-only alternative (or extension of) the experimentr framework?

experimentr. L. Harrison, C. Gramazio, F. Yang, K. Aragam, E. Peck, and D. Schroeder (2019).
github.com/codementum/experimentr
The first of which is a framework for running mobile-only visualization experiments,

The three projects I spoke about until now in this talk share a common infrastructure:

They're all node applications running as Azure web applications

With media queries and to enforce mobile-only participation

Server-side condition assignment and rebalancing for crowdsourced deployment

And handlers for orientation changes.

Refactoring this into an application framework could allow for experiment application developers to focus on

Just designing the stimuli, meaning the visualization components,

And defining what constitues a trial and how a bank of trials are initialized.

I should mention that there is a precedent for this.

A team led by Lane Harrison at WPI have developed a framework for crowdsourced visualization studies,

Though it is intended for desktop environments.

What I propose could be a mobile-only alternative or extensions to this,

And I believe that this would be a good intern project.
